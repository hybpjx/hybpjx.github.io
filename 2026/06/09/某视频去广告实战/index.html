某视频 去广告实战

前言

包名 com.lightmemory.simon

壳： 仅对单个 SDK（com.pp.hls）做了 360 加固保 SDK 级保护，dex 本体未整体加固

So层在标题4

广告点

首页 导航 开屏

首先找到了几个定位的位置

一共共有四处 分别是：

AcSplashAd / AcInterstitialAd / AcRewardAd / AcBannerAd

这是其中一部分的广告。全部都要hook掉

内链

还有一部分。是内联的一部分广告 com.moqi.sdk

ai和我说这是 MoQi 陌奇的SDK

ad_source_id == 4是 MoQi（陌奇）开屏，走SplashStartTdActivity→MQSplashAdView，完全不查 jd3.R()`，也不走 adcash.

播放页广告

逐个 hook 入口能拦广告，但是还有播放页的广告

也需要给他Hook掉

先找到 activity 然后再找到播放的位置逐一hook

这里还有一些开关

意义如下图。但是只hook这些位置 是远远不够的

方法 SP Key 含义
jd3.R() KEY_PREF_LOAD_AD_SHOW 广告总开关（==1 才放广告）
jd3.I() KEY_PREF_IS_FREE_AD_ALL 全天免广告标志
jd3.F() KEY_PREF_FREE_AD 免广告标志（播放页门控用）
jd3.U() KEY_PREF_MAXVIEWNUM 需观看视频广告总数（默认 5）
jd3.X() KEY_PREF_PLAYVIEWNUM 已观看数

只要让 jd3.R() 返回 0，App 自带的「无广告」分支就会生效，所有广告位自动跳过。 但是这是开发这告诉你的广告位跳过。实际上广告还是多成一坨。

看广告解锁 / 全天免广告

这个随便Hook Hook就好了 但是hook的都只是页面层的实际没什么卵用

最终实际要调用we2.a().b() 才能主动写入 这里直接写入999个

反检测：360 加固的 native 签名校验

用 NPatch集成模块后，App 开屏就崩。抓 logcat：

关键信息：

  • libjiagu_sdk_pp_hlsProtected.so —— 名字里的 jiagu 就是「加固」
  • JNI_ERR returned from JNI_OnLoad —— 这个 so 在加载时主动返回错误
  • 触发场景是运行 cache/npatch/origin/...apk（NPatch 释放原 APK 动态加载）

到 jadx 里搜相关类，找到：

com.qihoo.SdkProtected.pp_hls 表明这是 360 加固保对单个 SDK（pp_hls，一个 P2P 视频加速库）做的 SDK 级加固，而不是对整个 App 加壳。

这里也实际引用了这个So

这里我试了很多方法

1、删掉校验 so 加载

2、阉割 P2P 调用点

3、直接改smail （可以 但是仅限几个）

4、删 so

事实是都不太行。这里直接看 So

看了一眼 全是加固
